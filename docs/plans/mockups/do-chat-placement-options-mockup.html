Recommended: Dedicated Page

The current plan (Do as top-level nav item) gives full-screen real estate. No form/doc conflicts. These options are alternatives for A/B comparison if inline chat proves more natural.

📐

Form Space Is The Key Tradeoff

All inline options struggle with the working document pattern (goal forms, check-in forms) which needs ~50% screen width. The dedicated page avoids this entirely.

🔀

Hybrid Is Possible

Option C (side panel) with an "expand" button is the closest hybrid — quick chats stay inline, heavy workflows expand to full page. More complex to build but most flexible.
